# ANSVK Universal Component Master List

This document contains the locked, universal baseline of UI components for both the internal ERP (clinical governance/inventory) and the public-facing landing page. These component names will serve as the exact rows in the `Component_Variables` database table.

## 1. Core Primitives (Atomic Blocks)
* **AppButton:** Standard actions (Primary, Secondary, Outline).
* **AppIconButton:** Icon-only actions.
* **AppTextLink:** Clickable text for inline navigation.
* **AppBadge:** Status indicators (Active, Pending).
* **AppAvatar:** User or author image circle.
* **AppDivider:** Structural lines.

## 2. Data Entry (Form Controls)
* **AppTextField:** Standard inputs (Text, Number, Email, Password).
* **AppDropdown:** Select menus.
* **AppDatePicker:** Calendar selection.
* **AppCheckbox:** Boolean toggles/multi-select.
* **AppSwitch:** On/Off toggles.
* **AppFileUpload:** Document/image attachers.
* **AppRatingInput:** Star/scale input for reviews or clinical scales.

## 3. Layout & Containers
* **AppScaffold:** The main page wrapper.
* **AppCard:** Standard container with background, radius, and shadow.
* **AppGrid:** Wrapper for feature lists or dashboard metrics.
* **AppHorizontalScroll:** Wrapper for galleries or wide data.
* **AppDialog:** Modal popups.

## 4. Navigation (Global)
* **AppTopBar:** Header with title and profile.
* **AppSidebar:** ERP module switcher.
* **AppFooter:** Landing page footer (copyright, links).
* **AppSystemFooter:** ERP sticky footer (app version, user).
* **AppBreadcrumbs:** Page hierarchy trail.

## 5. Complex Data Display (ERP Focused)
* **AppTable:** The core data grid for inventory/HR.
* **AppListGroup:** Framed, dense lists for directories.
* **AppProfileHeader:** Standardized header for patient/employee records.
* **AppAttachmentGallery:** Viewer for receipts or certificates.

## 6. Marketing Blocks (Composites)
These are larger structures built strictly by assembling smaller components (Primitives, Containers, Form Controls). In the UI database, these items only require layout and spacing variables (e.g., `hero_section_padding`), as the inner components (like buttons and text) read their own respective database tokens.
* **AppHeroSection:** Layout Container + Typography + AppButton.
* **AppTestimonialCard:** AppCard + AppAvatar + Typography.
* **AppFeatureGrid:** AppGrid + AppCard + Icons + Typography.
* **AppContactForm:** AppCard + AppTextField + AppButton.

## 7. System & Feedback
* **AppSnackbar:** Temporary toast alerts.
* **AppSyncIndicator:** Online/Offline status indicator for offline-first architecture.
* **AppLoading:** Spinners or skeleton screens.
